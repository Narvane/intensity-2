# Intensity — Diretriz de Estilo Visual

## Conceito

O produto deve transmitir:

- diversão
- descoberta
- espontaneidade
- amizade
- conexão humana
- expectativa positiva

O usuário deve sentir que está abrindo uma caixa de surpresas, não utilizando uma ferramenta de produtividade.

---

# Personalidade

O visual deve ser:

- amigável
- leve
- otimista
- acolhedor
- energético
- casual

Evitar:

- aparência corporativa
- aparência bancária
- aparência SaaS B2B
- aparência de dashboard
- aparência de rede social

---

# Direção de Design

## Flat Cartoon UI

Características:

- formas simples
- cores sólidas
- poucos detalhes
- ilustrações minimalistas
- visual lúdico
- componentes grandes e acessíveis

Evitar:

- realismo
- glassmorphism
- skeuomorfismo
- excesso de gradientes
- excesso de sombras

---

# Cores

## Filosofia

Cada caixa deve parecer uma categoria divertida.

As cores ajudam na identificação.

## Paleta Principal

### Coral

`#FF6B3D`

Cor principal.

### Amarelo

`#FFC94D`

Descoberta.

### Roxo

`#7B5CF6`

Criatividade.

### Turquesa

`#2DBD9A`

Aventura.

## Neutros

### Fundo

`#FFF7ED`

### Branco

`#FFFFFF`

### Texto

`#1F1F1F`

### Texto Secundário

`#5A5A5A`

---

# Tipografia

## Estilo

Fontes arredondadas.

Sensação humana.

Exemplos:

- Nunito
- Nunito Sans
- Quicksand
- Rubik
- Plus Jakarta Sans

## Títulos

Peso:

`700–800`

Grandes e impactantes.

## Texto

Peso:

`400–500`

Muito legível.

---

# Bordas

Extremamente arredondadas.

Botões:

`24px`

Cards:

`20px`

Chips:

`999px`

---

# Linhas

Mínimas.

Preferir:

- espaço
- agrupamento
- contraste

Ao invés de separadores visuais excessivos.

---

# Sombras

Muito suaves.

```css
box-shadow: 0 4px 12px rgba(0,0,0,.06);
```

Sombras servem apenas para separar camadas.

Nunca para criar realismo.

---

# Ilustrações

## Estilo

Flat cartoon.

Características:

- traço simples
- poucos detalhes
- expressões felizes
- formas arredondadas

## Temas

- pessoas conversando
- viagem
- café
- praia
- jogos
- aventura
- amizade

## Evitar

- cenas realistas
- pinturas
- fotografias
- renderizações 3D

---

# Ícones

Estilo:

- outline simples
- cantos arredondados
- espessura consistente

Exemplos:

- Lucide
- Heroicons
- Phosphor

---

# Componentes

## Cards

Devem parecer "caixinhas colecionáveis".

Características:

- cor sólida
- ícone grande
- título curto
- contador de ideias

Exemplo:

```text
┌─────────────┐
│ ☕          │
│             │
│ Sair da     │
│ Rotina      │
│             │
│ 31 ideias   │
└─────────────┘
```

---

# Animações

Sensação de jogo casual.

Pequenas animações:

- bounce
- scale
- confetti
- reveal

Evitar:

- transições lentas
- animações dramáticas
- motion excessivo

---

# Sorteio

Este é o centro emocional do produto.

Deve parecer:

- abrir um envelope
- revelar uma carta
- descobrir um prêmio

Nunca:

- uma lista aleatória
- um modal comum
- um sorteador técnico

---

# Regra Principal

Se uma tela parece algo que poderia existir em:

- Jira
- Trello
- Notion
- Monday
- Asana

então está errada.

Se parece algo que poderia existir em:

- Duolingo
- Headspace
- Finch
- um jogo mobile casual

então provavelmente está na direção certa.

---

# Resumo Executivo

**Flat cartoon social adventure UI: cores vibrantes, formas arredondadas, ilustrações simples, clima de descoberta e sensação de jogo casual, sem qualquer aparência corporativa ou produtiva.**
